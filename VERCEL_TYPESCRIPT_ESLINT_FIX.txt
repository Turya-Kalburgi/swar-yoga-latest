PROBLEM:
  24 TypeScript TS2307 "Cannot find module 'uuid'" errors during Vercel build
  Errors were in server models trying to import uuid
  Persisted even after adding uuid to root package.json

ROOT CAUSE:
  ESLint was running eslint . which matched ALL .ts/.tsx files
  This included server/**/*.ts files
  ESLint tried to type-check server files
  Server TypeScript compilation couldn't find uuid in build context

SOLUTION:
  1. Restrict ESLint to only check src/ folder
  2. Add server/ to TypeScript exclude lists
  3. Configure Vercel to ignore non-frontend folders
  4. Remove API route rewrites that might trigger confusion

LESSON #1: Frontend Build Tools â‰  Backend Build Tools

  Don't include backend files in build/lint configuration

LESSON #2: ESLint Patterns are Powerful and Dangerous
  files: ['**/*.{ts,tsx}'] matches EVERYTHING
  files: ['src/**/*.{ts,tsx}'] is safer and more specific

LESSON #3: Build Warnings vs Errors
  Previous fix added uuid to root, which helped
  But errors persisted because root was linting server files
  Had to address the root cause: why are server files being linted?
